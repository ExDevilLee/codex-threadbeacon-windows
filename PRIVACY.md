# 隐私说明

## 数据范围

ThreadBeacon 只在本机读取以下 Codex 数据：

- `state_5.sqlite` 中最近未归档主任务，以及按本机已保存收藏 ID 指定的主任务（可包含已归档任务）的 ID、标题、更新时间、归档状态、模型、Reasoning effort、累计 Token、rollout 路径和直接父子关系；
- 用户展开主任务时，该主任务直接 Subagent 的标题、昵称、角色、模型、Reasoning effort、更新时间、累计 Token 和 rollout 路径；
- `session_index.jsonl` 中与当前主任务或已展开直接 Subagent ID 对应的最新 rename 名称；
- 对应 rollout JSONL 文件尾部最多 2 MiB 中的事件类型、时间戳、`turn_context` 模型/effort 和 Token 数字字段，用于推导状态、显示任务详情，以及确认最近 120 秒内更新的直接 Subagent 是否仍在运行。
- `logs_2.sqlite` 中仅限当前可见主任务 ID、且目标为 `codex_http_client::default_client`、`codex_core::responses_retry` 或 `codex_core::session::turn` 的 HTTP 200/400/429/503、重试进度、最终 Turn error 和固定模型容量错误文本，用于识别服务异常 episode。

App 不提取用户消息、助手回答正文、reasoning summary、命令、工具输出、文件内容或完整请求，也不读取第二层及更深的子任务。`codex_http_client::transport` 明确不在日志白名单中，因为它可能包含完整请求上下文。

## 数据处理

- SQLite 使用短生命周期、无连接池、只读连接，并启用 `query_only`。
- 日志正文只在单次刷新查询和解析期间短暂存在；任务快照仅保留 episode ID、异常分类、可选 HTTP 状态码、重试次数/上限、阶段和时间，不保留原始日志正文。
- 数据源健康诊断只在内存中保留四类固定状态、Rollout 成功/失败读取数量和最近一次成功刷新时间；不保留路径、任务 ID、标题、日志正文或原始异常，也不保存健康历史。
- 数据只在当前进程内存中用于生成界面状态。
- App 不上传任务数据、不启动网络服务，也不写入或修改 SQLite、session index 或 rollout 文件。用户明确开启自动恢复后，App 会通过 Windows UI Automation 控制已安装的 Codex App，仅向匹配任务提交用户配置的续接提示；ThreadBeacon 不直接调用 Codex 网络 API。
- “关于”窗口不会自动打开外部页面；只有用户主动点击 GitHub、Releases、隐私说明、MIT License 或支持项目按钮时，才会由默认浏览器打开仓库内固定公开链接。
- 启动检查更新只请求仓库固定的公开 GitHub Releases API，不上传任务标题、会话 ID、Token、日志正文、本机路径或其他任务数据；更新失败时不会重试任务数据或改变 Codex 文件。
- 当前可见且未归档的主任务只查询最近 120 秒内更新的直接 Subagent 候选并读取其 rollout 尾部；只有展开主任务时才读取直接 Subagent 的完整详情记录，且同轮刷新复用候选观察结果。
- Subagent 展开状态和子任务元数据不会写入设置或其他持久化文件。
- App 仅在本地保存窗口置顶状态、主窗口显示器设备名与数字位置/尺寸、刷新间隔、最大任务显示数量、提示音设置、最多 256 个派生完成/异常事件 ID，以及用户主动开启的登录启动注册表值；不保存任务标题、正文、Token 详情或 Codex 路径。
- 自定义提示音只保存用户主动选择的本机 WAV 文件路径；文件内容不会复制、上传或写入 Codex 数据目录，路径失效时自动回退对应内置音色。
- 任务级置顶与忽略规则只在本地保存任务 ID、忽略时间和规则类型，不保存任务标题；临时忽略会在检测到更晚的 `task_started` 后自动删除。
- 任务收藏只在本地保存任务 ID 集合和“仅显示收藏”布尔值，不保存标题或 Codex 路径。归档收藏不参与 429/503 日志查询，也不会生成完成或异常通知事件。

## 自动恢复数据

- 自动恢复默认关闭；App 启动、手动刷新和恢复监听只建立故障基线，不补发历史异常。
- 本机设置文件保存总开关、各异常开关和用户配置提示词；提示词最长 500 字。
- 本机历史最多保存 100 条任务 ID、故障 episode ID、异常分类、发送状态与时间，不保存提示词、任务标题、rollout 路径、日志正文或异常原文。
- 发送前要求唯一 Codex 窗口、空 composer、任务 ID 深链、精确 Rename 标题和唯一结构化发送按钮；发送只调用一次，确认超时不会重试。

## 已知边界

Codex 本地文件格式不是稳定的公开 API，未来版本可能改变表、字段、日志格式或路径。数据缺失、数据库繁忙或格式不兼容时，ThreadBeacon 会显示降级状态或空结果；日志源故障只会暂时停用异常识别，不会让主任务列表失败，也不会尝试修复或改写源数据。
