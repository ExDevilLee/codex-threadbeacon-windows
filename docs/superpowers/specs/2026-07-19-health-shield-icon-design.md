# 数据源健康小盾牌图标设计

## 目标

将主窗口右下角的数据源健康入口与 macOS 版本对齐。整体状态为正常时，入口显示绿色的“小盾牌 + 对钩”；部分降级和不可用仍分别显示橙色警告图形与红色错误图形。

## 范围

- 只调整主窗口右下角数据源健康入口的整体状态图标。
- 保留现有 22×22 点击区域、位置、颜色、提示文字、无障碍名称和弹窗行为。
- 弹窗内四个数据源的逐行状态图标保持不变。
- 不引入图片、SVG、第三方图标库或新的运行时依赖。

## 视觉映射

| 整体状态 | Windows 图标 | 颜色 | macOS 对应图标 |
| --- | --- | --- | --- |
| 正常 | Fluent `Shield` 轮廓叠加 `CheckMark` | 绿色 | `checkmark.shield` |
| 部分降级 | 现有警告图形 | 橙色 | `exclamationmark.triangle.fill` |
| 不可用 | 现有错误图形 | 红色 | `xmark.octagon.fill` |

正常状态使用 Windows 11 自带的 `Segoe Fluent Icons`。盾牌采用 `EA18`，对钩采用 `E73E`，在固定尺寸容器内居中叠放，以获得接近 SF Symbols `checkmark.shield` 的含义，同时保留 Windows 原生线条风格。

## 实现边界

`DataSourceHealthViewModel` 继续负责把整体状态映射为图标信息和颜色。健康状态额外提供盾牌底图与对钩叠加层；降级和不可用状态继续显示单个图标。`DataSourceHealthControl` 只负责在固定大小的图标区域内渲染这些属性，不改变弹窗的打开和关闭逻辑。

图标映射必须由状态确定，刷新只更新绑定值，不重新创建按钮或改变布局，因此不会影响两秒自动刷新、焦点或弹窗稳定性。

## 测试与验收

- 先增加 ViewModel 测试，证明健康状态暴露盾牌与对钩，且降级和不可用不显示叠加层。
- 增加或调整 XAML 测试，证明入口包含固定尺寸的双层图标容器，并绑定正确属性。
- 运行全部单元测试和 Release 构建。
- 启动应用，确认健康状态为正常时右下角显示绿色小盾牌对钩，点击后弹窗正常显示，进程不崩溃。
- 确认未引入新依赖、网络访问、数据写入或敏感信息。

## 非目标

- 不改变各数据源行内图标。
- 不重新设计底部状态栏或数据源健康弹窗。
- 不把 macOS 的 SF Symbols 资源直接复制到 Windows。
