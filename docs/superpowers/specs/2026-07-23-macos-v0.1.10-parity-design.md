# macOS v0.1.10 Parity Design

## Goal

Bring the five runtime deltas recorded through macOS commit `d18a18c` to the
Windows WPF implementation without changing the read-only Codex data boundary.

## Delivery Units

1. Accept `status: NNN` in both the in-memory incident parser and the SQLite
   prefilter.
2. Add a persisted per-task, per-incident automatic-recovery circuit breaker.
   Count distinct episodes, default to three attempts, allow limits from 1 to
   20, expose open states and manual reset, and reset after a newer confirmed
   completion.
3. Permit unattended deep-link navigation from a frontmost, unconfirmed Codex
   task only when its unique readable composer is empty. Preserve all existing
   target, draft, focus, runtime-ID, and send-time fail-closed checks.
4. Add a persisted 1-5 minute just-completed retention setting. Apply it to
   primary tasks and Subagents and refresh the baseline without replaying
   notifications.
5. Default color-blind-safe status symbols to enabled only when the setting is
   absent. Preserve every explicit saved true or false value.

## Architecture

Keep policy and persistence independent of WPF. Parser and retention behavior
remain in `ThreadBeacon.Core`; circuit-breaker state and policy use focused core
models plus an app-level JSON store; WPF view models expose only display-ready
options and commands. Existing Windows UI Automation remains the only Codex
control path.

## Safety

- Codex SQLite and rollout files remain read-only.
- Automatic recovery never stores prompts, titles, paths, or UI text in circuit
  state.
- Navigation and send operations revalidate target identity, composer runtime
  ID, empty/draft state, and foreground ownership immediately before action.
- Invalid settings and state files fall back safely without breaking monitoring.

## Verification

Each delivery unit follows test-first development, then the full Release test
suite, a warning-free Release build, fixed-location installation, UI-level
verification of the changed behavior, sensitive-data scanning, and an
independent commit and push.
